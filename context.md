# AIDP 标注工作区优化 Demo

## 项目定位
这是一个嵌入 AIDP 标注平台的标注工作区组件 Demo，用于 AI 小二「主动触达-基线评测标注」场景。目标是将外包标注同学的单 Case 标注时间从 10 分钟降至 5 分钟以内，核心手段是：消灭跨系统跳转（Trace 信息内联）、消灭反复滚动（三栏联动布局）、减少重复劳动（机标预填+人工复核）。本 Demo 使用 mock 数据，页面需完整可独立运行。

## 目标用户
- 外包标注同学：按照 SOP 对 AI 小二主动触达的 Case 逐一评分，覆盖 6 个维度（商家数据、店铺诊断、任务诊断、沟通话术、Agent卡片、发送时机），每个维度满分 5 分，总计 30 分。

## 整体 UI 风格
- 参照 AIDP 平台现有风格：白色背景、#1966FF 蓝色主色调、灰色边框分隔、表格化信息展示
- 字体：系统默认字体，正文 14px，标题 16px 加粗，次要信息 12px 灰色
- 圆角卡片 8px、阴影 0 2px 8px rgba(0,0,0,0.06)
- 整体布局撑满可视窗口，不出现页面级滚动条（各区域内部可独立滚动）

## 页面结构

本 Demo 只有一个页面：**Case 标注详情页**。没有路由切换，没有独立列表页。页面采用三栏联动布局，取消原有的左侧 Case 列表侧边栏（原来侧边栏是无效信息且占用大量屏效）。

### 整体布局结构（从上到下、从左到右）

```
┌─────────────────────────────────────────────────────────────────┐
│                        顶部信息栏（固定，高度 56px）                │
├────────────┬────────────────────────────┬───────────────────────┤
│  左侧导航栏  │       中间主内容区            │     右侧标注操作栏     │
│  宽 180px   │       自适应剩余宽度          │      宽 340px         │
│  固定不滚动   │      内部可滚动              │     固定不滚动         │
│             │                            │                       │
│  高度 =      │      高度 =                 │     高度 =             │
│  视口-56px   │      视口-56px              │     视口-56px          │
└────────────┴────────────────────────────┴───────────────────────┘
```

## 各区域详细设计

### 一、顶部信息栏

#### 用途
展示当前 Case 的基础元数据，提供全局操作入口。

#### 布局
- 高度 56px，白色背景，底部 1px #E5E6EB 边框
- 左侧：Case 元信息
- 右侧：全局操作按钮组

#### 元素清单

**左侧元信息区（水平排列，各项之间用竖线 | 分隔，字号 13px）：**
- 「Case」标签 + 序号显示：如 "Case 3 / 10"，序号加粗
- 店铺ID：如 "店铺ID: 257269496"
- 诊断日期：如 "诊断日期: 2026-03-31"
- 触达渠道：如 "触达渠道: 抖店IM"
- AI小二目标：如 "目标: 报名商家成长任务"
- 整体进度指示：如 "评分进度: 2/6"，用小圆点图标表示每个维度状态（灰色=未处理，蓝色=已完成，红色=有扣分）

**右侧操作按钮区：**
- 「标记红牌 0分」按钮：红色描边按钮，点击弹出二次确认弹窗
- 「提交标注」按钮：蓝色实心主按钮，6 个维度全部确认后才可点击（否则置灰），点击后弹出确认弹窗显示总分摘要，确认后加载下一个 Case
- 「跳过」按钮：灰色文字按钮

#### 交互

- 点击「提交标注」→ 弹出确认弹窗，内容为：
  ```
  确认提交标注？
  ─────────────────
  商家数据：5分 ✅
  店铺诊断：5分 ✅
  任务诊断：3分 ⚠️（行动点优先级排序错误 -2）
  沟通话术：5分 ✅
  Agent卡片：5分 ✅
  发送时机：5分 ✅
  ─────────────────
  总分：28 / 30
  [取消] [确认提交]
  ```
- 确认提交后 → 页面内容切换为下一个 Case（mock 数据切换，带一个 300ms 的加载过渡动画），顶部序号更新
- 点击「标记红牌 0分」→ 弹出确认弹窗："确认将此 Case 标记为红牌（0分）？此操作将跳过所有维度评分。" → 确认后跳转下一个 Case
- 点击「跳过」→ 直接加载下一个 Case

### 二、左侧导航栏

#### 用途
展示 6 个标注维度的导航入口和实时状态，以及红牌预检结果汇总。

#### 布局
- 宽度 180px，白色背景，右侧 1px #E5E6EB 边框
- 高度 = 视口高度 - 56px（顶部栏高度），固定定位不随任何区域滚动
- 内部分为三个区块：维度导航列表、红牌预检区、底部操作区

#### 元素清单

**维度导航列表（顶部，占据主要空间）：**

6 个维度按固定顺序垂直排列，每个维度是一个可点击的导航项：

| 序号 | 维度名称 | 说明 |
|------|---------|------|
| 1 | 商家数据 | 满分5分 |
| 2 | 店铺诊断 | 满分5分 |
| 3 | 任务诊断 | 满分5分 |
| 4 | 沟通话术 | 满分5分 |
| 5 | Agent卡片 | 满分5分 |
| 6 | 发送相关 | 满分5分 |

每个导航项的结构：
- 左侧：状态图标（圆形，直径 8px）
  - 灰色空心圈 ○：未处理
  - 蓝色旋转图标 🔄：机标已预填，等待确认
  - 绿色实心勾 ✅：已确认，满分
  - 橙色实心勾 ✅：已确认，有扣分（非底线）
  - 红色叹号 ❌：已确认，命中底线问题
- 中间：维度名称（14px）
- 右侧：得分显示（仅已确认后显示，如 "5分" 或 "0分"）
- 当前选中的维度：左侧加 3px 蓝色竖条指示，背景色 #F2F7FF

**红牌预检区（中部，与导航列表之间有 16px 间距 + 分隔线）：**
- 标题："红牌预检" 字号 12px 灰色
- 预检结果：
  - 全部通过 → 绿色图标 + "全部通过" 绿色文字
  - 有疑似问题 → 红色图标 + "发现 X 项疑似底线" 红色文字，点击可展开查看具体项

**底部操作区：**
- 显示当前总分："当前总分：XX / 30"（实时更新）

#### 交互

- 点击某个维度导航项 → 中间主内容区切换到对应 Tab → 右侧标注操作栏切换到对应评分面板
- 当前选中态有蓝色左侧竖条 + 浅蓝背景
- 页面首次加载时，默认选中第一个维度「商家数据」

### 三、中间主内容区

#### 用途
按当前选中的维度，展示标注员做判断所需的全部信息。这是信息的核心展示区域，通过 Tab 切换避免长页面滚动。

#### 布局
- 宽度：自适应填充（视口宽度 - 180px - 340px）
- 高度：视口高度 - 56px
- 白色背景，左右各有 1px #E5E6EB 边框
- 内部可垂直滚动（当某个维度的信息较多时）
- 顶部无 Tab 栏（Tab 切换通过左侧导航栏点击触发，中间区域直接展示对应内容）
- 内容区内边距 24px

#### 各维度 Tab 内容详细设计

每个维度 Tab 的内容由 2-3 个卡片区块垂直排列组成：**机标预检结果卡片**（始终在最顶部）+ **信息展示卡片**（1-2个）。

### 四、右侧标注操作栏

#### 用途
固定展示当前选中维度的评分操作面板，标注员在此完成打分、选择错误标签、填写说明。

#### 布局
- 宽度 340px，白色背景，左侧 1px #E5E6EB 边框
- 高度 = 视口高度 - 56px，固定定位不随中间区域滚动
- 内部当内容较多时可独立滚动
- 内边距 20px

#### 元素清单（以「店铺诊断评估」维度为例，其他维度结构相同，仅错误标签不同）

**顶部：维度标题区**
- 维度名称：如 "店铺诊断评估"（16px 加粗）
- 满分说明：如 "满分 5 分"（12px 灰色）

## 交互流程总览

### 主流程（标注员完成一个 Case 的完整路径）

1. 页面加载 → 系统自动拉取 Trace 信息、会话时光机数据、商家真实数据 → 运行全部机标预检规则 → 页面渲染完成
2. 加载过程中显示骨架屏（各区域用灰色占位块），加载完成后过渡为实际内容（因为是 mock 数据 Demo，加载过程模拟 800ms 延迟展示骨架屏效果）
3. 默认进入第一个维度「商家数据」
4. 标注员逐维度完成评分：
   - 查看中间区的信息和机标结果 → 在右侧确认或修改评分 → 确认 → 自动跳转下一维度
5. 6 个维度全部确认 → 点击顶部「提交标注」→ 确认弹窗 → 确认 → 加载下一个 Case

### 异常流程

**空状态：**
- 如果 Trace 拉取失败（mock 场景不会出现，但需有占位）：中间区对应卡片显示 "Trace 信息加载失败，请检查诊断日志ID是否正确" + 「重试」按钮
- 如果会话时光机无触达记录：显示 "该商家暂无触达记录"

**加载状态：**
- 页面首次加载时显示骨架屏（800ms mock 延迟）
- 切换维度 Tab 时无需重新加载（数据已预先全部拉取）

**红牌操作：**
- 任何时刻点击左侧底部或顶部「标记红牌 0分」→ 弹窗确认 → 所有维度分数归零 → 跳转下一 Case

**提交校验：**
- 如果有维度未确认评分，点击「提交标注」→ 弹出提示 "还有 X 个维度未完成评分，请先完成所有维度的评分"，未完成的维度在左侧导航栏闪烁提示

**最后一个 Case：**
- 当提交的是最后一个 Case 时（mock 总共 3 个 Case），提交后显示 "所有 Case 已标注完成 🎉" 的完成页面，包含总览统计：已标注 3 个 Case，平均得分 XX 分

## 设计规范

### 色彩系统
- 主色调：#1966FF（蓝色，按钮、选中态、链接）
- 成功色：#00B365（绿色，✅通过状态）
- 警告色：#FF7D00（橙色，⚠️需关注状态）
- 错误色：#F53F3F（红色，❌错误/底线状态）
- 背景色：#FFFFFF（主背景）、#F7F8FA（次要背景/卡片间距）、#F2F7FF（蓝色浅背景/选中态）
- 边框色：#E5E6EB（分隔线、卡片边框）
- 文字色：#1D2129（主要文字）、#4E5969（次要文字）、#86909C（辅助文字/placeholder）

### 组件样式
- 按钮：高度 32px，圆角 4px，主按钮蓝色实心白字，次要按钮灰色描边，危险按钮红色描边
- 卡片：白色背景，1px #E5E6EB 边框，8px 圆角，阴影 0 2px 8px rgba(0,0,0,0.06)，内边距 16px
- 表格：表头 #F7F8FA 背景，行高 40px，hover 行背景 #F2F7FF
- Checkbox：16x16px，选中态蓝色实心
- 输入框：高度 32px，#E5E6EB 边框，圆角 4px，focus 态蓝色边框
- 标签：小圆角（2px），[底线🔴] 用红色背景白字，其他状态标签用对应颜色浅背景深字
- 弹窗：白色背景，居中显示，半透明黑色遮罩，宽度 480px，圆角 12px

### 文字规范
- 所有文案使用贴近真实业务的中文，不使用 Lorem ipsum 或 placeholder
- 店铺名称、金额、日期等数据使用真实格式的 mock 值
- 按钮文案简洁明确：「确认评分」「采纳建议」「手动评分」「提交标注」「标记红牌 0分」

### 响应式
- 本 Demo 仅适配桌面端（最小宽度 1280px）
- 不需要适配移动端
- 当视口宽度 < 1280px 时，中间主内容区保持最小宽度 600px，其他区域不缩小

### 动效
- Tab 切换：中间内容区淡入淡出 200ms
- 确认评分：左侧导航项状态图标有一个 300ms 的颜色过渡
- 提交后加载下一 Case：整个内容区域 300ms 淡出 → 骨架屏 800ms → 新内容 300ms 淡入
- 弹窗：300ms 缩放 + 淡入动画

## AI 行为约束

- 修改某个模块时，不要动其他已完成的模块
- 不要自己发明 UI 组件，使用原生 HTML + CSS 实现，或使用 React + 简单 CSS-in-JS
- 所有文案用贴近真实业务的中文，不要用 placeholder 文本
- 每次修改前先说明你要改什么、影响范围是什么
- 三栏布局必须严格固定：左侧导航栏和右侧标注栏不随中间内容区滚动
- 中间内容区的各维度 Tab 切换由左侧导航栏点击驱动，不要在中间区域顶部额外加 Tab 栏
- Mock 数据直接写在代码中（JSON 对象），不要做成外部 API 调用
- 页面必须能独立运行（单个 HTML 文件或 React 项目均可），不依赖外部后端服务
- 机标预检结果中的 ✅ ❌ ⚠️ 状态图标使用 emoji 或 SVG，不要依赖外部图标库 CDN
- 卡片渲染预览（Agent卡片维度）用 CSS 模拟 IM 卡片样式即可，不需要真实渲染引擎
- 骨架屏使用 CSS 动画实现灰色脉冲效果，不需要第三方库
- 表格中不一致/缺失的行必须有明显的视觉区分（红色/橙色背景高亮）
- 错误标签的层级结构（一级问题 → 二级问题）必须严格按照上述定义实现，不要自行增减
- 底线问题标签旁的 [底线🔴] 标记必须清晰可见
